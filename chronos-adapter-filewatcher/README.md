# chronos-adapter-filewatcher

The Filesystem Watcher Adapter is a Perception Layer subsystem. Its sole responsibility is to monitor configured local directories recursively for file creations, modifications, deletions, and moves, converting these observations into normalized `ChronosEvent` logs published onto the `Cognitive Bus`.

## Specification

### Consumes
- **OS Filesystem Notifications**: Raw events generated by the OS (using `ReadDirectoryChangesW` on Windows via the `notify` crate).

### Produces
- `FileCreated`
- `FileModified`
- `FileDeleted`
- `FileMoved`

### Capabilities
- **Recursive Directory Watching**: Listens to changes across entire folder structures.
- **Burst Debouncing**: Automatically groups multiple close modifications on the same file within a short time window (e.g. 500ms) to prevent event storming.
- **Metadata Extraction**: Extracts absolute paths, file extensions, and file sizes.

### Dependencies
- `chronos-core`: Event models.
- `chronos-bus`: Bus publication interfaces.
- `chronos-registry`: Service descriptor registration.
- `chronos-container`: Dependency injection support.
- `chronos-config`: Watch configuration paths.
- `chronos-logging`: Observability structures.
- `notify`: Platform-native monitoring wrapper.

### Failure Modes
- **Access Violation**: Errors are handled gracefully inside OS handlers and logged via `ChronosLogger`.
