# Chronos UI Migration Plan

## 1. Executive Summary
The current Chronos MVP frontend contains advanced concepts (Mission Control, Risk Analysis, Autonomous Research, Cognitive Forensics) that were rapidly prototyped with mock data and client-side logic. 

As the backend strictly adopts the 7-Layer Personal Context Operating System (PCOS) architecture, the frontend must transition from a "smart client" to a "dumb terminal" (Layer 6 — Interaction). All reasoning, decision-making, and state management must be migrated to their canonical owning layers in the Rust daemon.

This document serves as the mapping contract between the existing MVP screens and the frozen PCOS architecture.

---

## 2. Screen & Quadrant Mapping

### 2.1 Mode 01: Invisible Flight Recorder
*   **Purpose:** View the raw background telemetry and event feed.
*   **User Question Answered:** What is happening in the background? What did Chronos observe?
*   **Current Data Source:** SQLite telemetry logs (`/api/telemetry-logs`).
*   **Future Data Source:** The Cognitive Bus feeding from the Event Store (Observation level).
*   **Owning PCOS Layer:** **Layer 6 (Interaction)**. Data provided by **Layer 2 (Memory)** and generated by **Layer 1 (Perception)**.
*   **Consumes:** Stream of canonical `ChronosEvent`s.
*   **Produces:** Read-only view.
*   **Required Backend APIs:** `GET /api/events/stream` (via Cognitive Bus/Event Store).

### 2.2 Mode 02 / Quadrant 01: Risk Analysis (Failure Forecasting)
*   **Purpose:** Predict timeline decay and project failure probabilities.
*   **User Question Answered:** What is at risk?
*   **Current Data Source:** Mocked commitments array from `/api/commitments`.
*   **Future Data Source:** Personal Capacity Model (PCM) and Deadline Discovery Engine (DDE).
*   **Owning PCOS Layer:** **Layer 6 (Interaction)**. Data owned by **Layer 3 (Reasoning)**.
*   **Consumes:** Risk scores, decay trajectory coordinates, and Monte Carlo probabilities.
*   **Produces:** Manual Context Ingestion events.
*   **Required Backend APIs:** 
    *   `GET /api/reasoning/forecasts`
    *   `POST /api/perception/ingest` (Manual observation injection)

### 2.3 Mode 02 / Quadrant 02: Strategy Ledger
*   **Purpose:** Display active continuation plans and project execution tasks.
*   **User Question Answered:** What should I do next?
*   **Current Data Source:** Mocked commitments and actions (`/api/actions`).
*   **Future Data Source:** Context Continuation Engine (CCE) and Action Executor.
*   **Owning PCOS Layer:** **Layer 6 (Interaction)**. Data owned by **Layer 5 (Execution)**.
*   **Consumes:** Active commitments, generated recovery plans, priority indexes.
*   **Produces:** Task toggle intents, Plan generation requests.
*   **Required Backend APIs:** 
    *   `GET /api/execution/commitments/active`
    *   `POST /api/execution/generate-recovery-plan`
    *   `POST /api/execution/toggle-action`

### 2.4 Mode 02 / Quadrant 03: Cognitive Forensics
*   **Purpose:** Explain context blockage and reasons for workflow abandonment.
*   **User Question Answered:** Why did I stop?
*   **Current Data Source:** Static/LLM narrative from `/api/system-status`.
*   **Future Data Source:** Reflection Engine and Meta-Cognition Engine.
*   **Owning PCOS Layer:** **Layer 6 (Interaction)**. Data owned by **Layer 3 (Reasoning)** and **Layer 4 (Decision)**.
*   **Consumes:** Explainability narratives, blockage diagnostics.
*   **Produces:** Read-only view.
*   **Required Backend APIs:** `GET /api/reasoning/diagnostics`

### 2.5 Mode 02 / Quadrant 04: State Resumption
*   **Purpose:** One-click restoration of dormant projects.
*   **User Question Answered:** How do I get back into flow?
*   **Current Data Source:** Basic projects list (`/api/projects`).
*   **Future Data Source:** Memory layer (Project State) and Context Continuation Engine (Execution).
*   **Owning PCOS Layer:** **Layer 6 (Interaction)**. Action executed by **Layer 5 (Execution)**.
*   **Consumes:** Project context bundles (IDE paths, browser tabs, terminal histories).
*   **Produces:** Workspace restoration intents.
*   **Required Backend APIs:** `POST /api/execution/restore-workspace`

### 2.6 Mode 02 / Quadrant 05: Autonomous Research Crawler (ARC)
*   **Purpose:** Display background intelligence gathering.
*   **User Question Answered:** What is Chronos researching for me?
*   **Current Data Source:** Client-side static mock components.
*   **Future Data Source:** ARC Engine.
*   **Owning PCOS Layer:** **Layer 6 (Interaction)**. Data owned by **Layer 5 (Execution)**.
*   **Consumes:** Active background tasks, scraped knowledge summaries.
*   **Produces:** Read-only view.
*   **Required Backend APIs:** `GET /api/execution/arc-status`

### 2.7 Mode 03: Intervention Decision Engine (Simulator)
*   **Purpose:** Mathematical simulation of distraction vs. context debt.
*   **User Question Answered:** When should Chronos forcefully interrupt me?
*   **Current Data Source:** Hardcoded math in React (`App.tsx`).
*   **Future Data Source:** Decision Orchestrator.
*   **Owning PCOS Layer:** **Layer 6 (Interaction)**. Mathematical logic belongs to **Layer 4 (Decision)**.
*   **Consumes:** User simulation parameters (Days Untouched, Drift, Focus Level).
*   **Produces:** Simulated decision outcomes.
*   **Required Backend APIs:** `POST /api/decision/simulate`

## 3. UI Component Classification

| Component | Decision | Rationale |
| :--- | :--- | :--- |
| **Global HUD Banner** | **KEEP** | Maps perfectly to Layer 6 telemetry needs. Requires streaming API for live CPU/Gatekeeper stats. |
| **Mode Selector** | **KEEP** | Perfectly delineates the 3 core visual interaction states of the OS. |
| **FlightRecorderPanel** | **MODIFY** | Rip out direct SQLite querying. Subscribe to Cognitive Bus stream via a new API endpoint. |
| **RiskForecaster** | **KEEP** | Visuals remain, but data must be wired to Layer 3 (Reasoning) APIs instead of mock objects. |
| **Manual Ingestion Modal** | **MODIFY** | Needs to map to a standardized Perception Adapter API (acting as a "Manual UI Adapter" in Layer 1). |
| **CommitmentList** | **KEEP** | Shift AI plan generation from direct LLM endpoint to Layer 5 (Context Continuation Engine) execution request. |
| **Cognitive Forensics** | **KEEP** | Awaiting Layer 3 Reflection Engine implementation. |
| **WorkspaceRestorer** | **KEEP** | Requires actual execution logic in Layer 5 to open processes and arrange windows on the host OS. |
| **ARCPanel** | **KEEP** | Awaiting ARC backend engine in Layer 5. |
| **Theory Simulator** | **MODIFY** | Remove local React math. Send slider states to backend Layer 4 Decision Orchestrator simulation API. |

## 4. Identified Implementation Gaps (Backend Capability Requirements)

To fully migrate the frontend away from mock data, the following backend capabilities must be implemented in future phases:

1.  **Semantic Entity Resolution (Layer 2 - Memory)**
    *   *Need:* The ability to group raw events into "Artifacts", "Concepts", and "Commitments" so the Strategy Ledger can display them.
2.  **Context Restoration Automation (Layer 5 - Execution)**
    *   *Need:* Cross-platform (or Windows-specific) capabilities to physically launch IDEs, restore browser tabs, and reopen terminals.
3.  **Explainability Engine (Layer 3 - Reasoning)**
    *   *Need:* AI-driven summarization of *why* a project stalled (Cognitive Forensics) and *why* an action is recommended (Priority Index diagnostics).
4.  **Decision Orchestrator Math (Layer 4 - Decision)**
    *   *Need:* The Silence Cost vs. Interruption Cost equation must be formalized in Rust so the HUD and the background daemon share the exact same logic.
5.  **Recovery Plan Synthesis (Layer 5 - Execution)**
    *   *Need:* The backend must accept a "RED" commitment and generate a step-by-step recovery trajectory via the AI Gateway.
